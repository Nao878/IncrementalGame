## 1. ゲーム概要

### 1.1 コンセプト
2D トップダウン視点のファクトリー型インクリメンタルゲーム。  
プレイヤーはグリッド上にコンベアや施設を配置し、**漢字が書かれたブロック**を搬送・合体させて新たな漢字を生み出し、コインを稼ぐ。

### 1.2 コアループ

```
[ジェネレーター] ──(漢字ブロック生成)──▶ [コンベア搬送] ──▶ [合体 or 回収]
       ▲                                                         │
       │                 コインを獲得してさらに施設を建設              │
       └──────────────────────────────────────────────────────────┘
```

1. **生成**: ジェネレーター (ItemGenerator) が一定間隔で漢字ブロックを生成する
2. **搬送**: コンベアベルト (ConveyorBelt) がブロックをグリッド上でティック駆動で移動させる
3. **合体**: 同一セルに2つのブロックが収束すると、レシピテーブルに基づいて合体判定 → 成功なら新漢字ブロックに変化、失敗なら「詰まり（Clog）」ブロックになる
4. **回収**: コレクター (CollectorNode) がブロックを吸収し、画数に応じたコインを獲得
5. **建設**: 獲得コインでコンベア・合成機・回収機を追加配置 → ラインを拡張

### 1.3 ゲームモード

| モード | Time.timeScale | 操作 |
|--------|---------------|------|
| **Build Mode（編集モード）** | `0`（時間停止） | 施設の配置・回転・キャンセル |
| **Run Mode（稼働モード）** | `1`（通常速度） | ラインが稼働、ブロック搬送・合体が進行 |

- ゲーム起動時は **Build Mode** で開始
- 画面右上のボタンで切り替え
- Run Mode 中は施設選択が自動解除される

## 3. エンティティ一覧

### 3.1 アイテム（Item / ItemData）

ブロックとしてコンベア上を流れるゲームの中心オブジェクト。

| プロパティ | 型 | 説明 |
|-----------|-----|------|
| `ItemText` | string | 表示する漢字テキスト (例: "木", "林") |
| `ItemColor` | Color | ブロックの背景色 |
| `MergeCount` | int | 合体回数（色の段階決定に使用） |
| `IsClog` | bool | 詰まりブロックか否か |

#### 移動ロジック
- `moveSpeed = 4.0` (ワールド単位/秒)
- SmoothStep による滑らかな補間移動 (`t * t * (3f - 2f * t)`)
- 移動完了後、到着先セルに施設がある場合は `FacilityNode.OnItemArrived()` が呼ばれる

#### 合体成功演出
- `PlayMergeSuccessPop()`: サイン波によるスケールアニメーション（0.22秒、最大1.38倍）
- `MergeEffectPlayer.PlayMergeBurst()`: パーティクルバースト（32粒、白→金→オレンジのグラデーション）

#### 詰まり (Clog) ブロック
- 色: `(0.5, 0.22, 0.22, 1)` — 暗い赤茶色
- コンベアで移動しない（`IsClog` チェックで除外）
- ラベルの上に赤い "!" マークが表示される
- **右クリック** でプレイヤーが手動撤去可能

| ジェネレーター | 位置 | テキスト | 色 | 出力方向 |
|---------------|------|---------|-----|---------|
| Gen1 | (1, 8) | "木" | 赤系 (0.9, 0.4, 0.3) | Right |

- セルにアイテムがある場合は生成をスキップ（詰まり防止）
- ジェネレーターのセルにもコンベアが自動配置される（出力方向矢印表示）

### 3.3 コンベアベルト（ConveyorBelt）

| パラメータ | デフォルト値 | 説明 |
|-----------|-------------|------|
| `speed` | 2.0 | 搬送速度（**要確認**: 現在この値はアイテム移動速度には直接使われていない模様） |

- 4方向: `Up`, `Right`, `Down`, `Left`
- 矢印スプライト（緑色）でコンベアの方向を視覚的に表示
- 既存マスに再配置するとローテーション（方向を時計回り90°回転）

### 3.4 施設ノード（FacilityNode 継承ツリー）

```
FacilityNode (abstract)
  ├── CombinerNode  — 合成機
  └── CollectorNode — 回収機
```

#### CombinerNode（合成機）
- 入力: 任意の方向からアイテムを受け入れ（バッファに蓄積）
- 合体判定: バッファに2つ以上溜まったら FIFO で先頭2つを取り出し、`KanjiDatabaseManager.TryMergeKanji()` で合体
- 出力: `outputDirection` 方向の隣接セルに結果ブロックを生成
- 出力先にアイテムがある場合は合体保留
- ビジュアル: "+" テキスト（オレンジ色）と黄色の出力方向矢印

#### CollectorNode（回収機）
- アイテムが到着すると吸収・破壊
- コイン獲得量: `画数 × 10`（最低10コイン）
- ビジュアル: "✓" テキスト（緑色）

## 5. コアシステム

### 5.1 ティックシステム（ItemManager）

| パラメータ | デフォルト値 | 説明 |
|-----------|-------------|------|
| `tickInterval` | 0.8秒 | 1ティックの間隔 |

#### ティック処理フロー
1. 全アイテムのリストから null を除去
2. 各アイテムについて:
   - 移動中・合体待ち・Clog のアイテムはスキップ
   - 現在セルにコンベアがなければスキップ
   - コンベアの方向に基づいて次のセルを計算
   - 次のセルが範囲外ならスキップ
   - 次のセルに Clog ブロックがあればスキップ
   - 次のセルに動いていないアイテムがあり、コンベアも施設もなければスキップ
3. 同一セルへの収束判定:
   - **1体のみ**: そのまま移動
   - **2体以上**: `MergeSequence` コルーチンを開始

### 5.2 合体判定（コンベア収束マージ）

コンベア上で2つのブロックが同じセルに収束した場合:

1. 全参加ブロックを移動させ、到着を待つ
2. 生存ブロックが2つでない場合 → **Clog** 生成
3. 「左から来たブロック」と「右から来たブロック」を判別
4. 左右のペアが揃わない場合 → **Clog** 生成
5. `KanjiDatabaseManager.TryMergeKanji([left, right])` で合体レシピ照合
6. レシピが存在しない場合 → **Clog** 生成
7. 成功時: 両ブロックを破壊 → 新ブロック生成 + パーティクル演出 + ポップアニメーション

> **重要**: 現在のコンベア収束マージは **左右方向のみ** を判定している（`pos + Vector2Int.left` / `pos + Vector2Int.right`）。上下からの収束はすべて Clog になる。

### 5.3 合体判定（CombinerNode 合成機マージ）

合成機ノード内のバッファベースマージ:

1. アイテムがノードに到着 → バッファに追加
2. バッファが2つ以上になると先頭2つで合体試行
3. レシピが見つかれば `outputDirection` 方向に結果ブロックを出力
4. **要確認**: レシピ不一致時の挙動（現在のコードではバッファに残り続ける → バグの可能性）

### 5.4 合体カラーシステム

アイテムの `MergeCount` に応じて色が変化（最大8段階）:

| MergeCount | 色 | 説明 |
|-----------|-----|------|
| 0 | (0.9, 0.4, 0.3) | 赤系（初期ブロック） |
| 1 | (0.3, 0.5, 0.9) | 青系 |
| 2 | (0.3, 0.85, 0.4) | 緑系 |
| 3 | (0.9, 0.8, 0.2) | 黄系 |
| 4 | (0.7, 0.3, 0.9) | 紫系 |
| 5 | (1.0, 0.6, 0.2) | オレンジ系 |
| 6 | (0.2, 0.9, 0.9) | シアン系 |
| 7+ | (0.95, 0.95, 0.95) | 白（最高ランク） |

## 12. 技術的メモ
