## 1. ゲーム概要

### 1.1 コンセプト
2D トップダウン視点のファクトリー型インクリメンタルゲーム。  
プレイヤーは空き地のグリッド上に発生機やコンベア、合成機などを配置し、**漢字が書かれたブロック**を搬送・合体させて新たな漢字を生み出し、中央のメインハブに納品してコインを稼ぐ。
序盤の資金ショートを防ぐため、初めてコインを獲得するまでは全施設が無料で配置できるインビジブル・チュートリアルが搭載されている。

### 1.2 コアループ

```
[ジェネレーター] ──(漢字ブロック生成)──▶ [コンベア搬送] ──▶ [合成機で合体] ──▶ [メインハブ納品]
       ▲                                                                   │
       │                       コインを獲得してさらに施設を建設                    │
       └───────────────────────────────────────────────────────────────────┘
```

1. **生成**: ジェネレーター (ItemGenerator) が一定間隔でランダムな基本漢字ブロックを生成する
2. **搬送**: コンベアベルト (ConveyorBelt) がブロックを方向指定で搬送する
3. **合体 (Shapez方式)**: 専用施設「合成機 (Combiner)」に左右両側から漢字ブロックを入力し、合体させて前方へ出力する
4. **納品 / 焼却**: メインハブ (Main Hub) にブロックを搬入して画数に応じたコインを獲得するか、不要な手荷物を焼却炉 (Incinerator) で破棄する
5. **建設**: 獲得コインでコンベア・発生機・合成機を追加配置し、生産効率を拡張する

### 1.3 ゲームモード

| モード | Time.timeScale | 操作 |
|--------|---------------|------|
| **Build Mode（編集モード）** | `0`（時間停止） | 施設の配置・回転・撤去 |
| **Run Mode（稼働モード）** | `1`（通常速度） | ラインが稼働、ブロック搬送・合体が進行 |

## 3. エンティティ一覧

### 3.1 アイテム（Item / ItemData）

ブロックとしてコンベア上を流れるゲームの中心オブジェクト。

| プロパティ | 型 | 説明 |
|-----------|-----|------|
| `ItemText` | string | 表示する漢字テキスト |
| `MergeCount` | int | 合体回数（色の段階決定に使用） |
| `IsClog` | bool | 詰まりブロックか否か（※現在廃止予定） |

- 【旧仕様】合成機での合成失敗時に発生し、警告テキスト "詰まり" が強調表示された。
- 新仕様では合体失敗時はそのまま「ゴミ（廃棄物）」扱いとしてコンベア上を流れるため、通常のゲームプレイでラインが完全に詰まることは廃止された。
- **削除モード（Delete Mode）** 中に左クリックすることで、施設と同様に漢字ブロックを直接削除できるようになった。
- 以前の「右クリックで手動撤去」機能も引き続き利用可能。

### 3.2 施設ノード（FacilityNode 継承ツリー）

#### ItemGenerator（発生機）
- 一定間隔(`spawnInterval` デフォルト3秒)で特定の基本漢字を生成。
- プレイヤーは配置前にUI（漢字指定ボタン）から生成する漢字（木、日、月、火、水、金、土 等）を事前に選択可能。
- 稼働モード（または配置後）に配置済みの発生機をクリックすることで、その場で生成する漢字をサイクル変更可能。
- 配置プレビュー（ゴースト）には選択中の漢字がリアルタイムで表示される。
- 生成される漢字テキストが上部に、出力方向を示すインジケーター（▲）が表示され、アイテム生成後、直ちに指定された排出方向のマスへブロックを押し出す。

#### CombinerNode（合成機）
- 出力ポート（前方）以外の3方向（後・左・右）に隣接するマスのコンベア上にある文字ブロックを自動的に検知し、能動的に内部スロットへ「吸い込む（Proximity Intake）」施設。
- 視覚的フィードバックとして、3つの入力予定ポートには内向きの青白い矢印、出力ポート（前方）には外向きの黄色い矢印が表示される。
- 取り込まれた文字は即座に施設上の2つの汎用テキストUI（Slot 1, Slot 2）に反映される（空の場合は「?」を表示）。
- 吸い込まれたアイテムは中央へ移動（Lerp）したのち消去（Destroy）される。移動中にアイテムが削除された場合の安全対策（Nullチェック）が施されている。
- スロットに2つの文字が揃った瞬間に合体を試みる。この際、入力された順序を問わず（A+B または B+A）`KanjiRecipe.csv` のレシピを参照する「順不同（オムニディレクション）」照合を行う。
- 【合体成功時】: 出力ポート（前方）から新たに生成した漢字アイテムを送り出し、スロットを空に戻す。
- 【合体失敗時】: その場で2つの文字を連結した通常ブロック（廃棄物）として即座に排出し、スロットを空に戻す。
- 既に2つのスロットが埋まっている場合や、排出先が詰まっている場合は新たな吸い込みを行わない。

#### MainHubNode（メインハブ）
- 2x2 マスを占有する巨大な固定配置施設（ゲーム開始時に自動生成される）。
- 表示されるラベルテキストは施設サイズ（RectTransform・Scale）に合わせて最適化され、画面全体を覆わないよう設定されている。
- 到達したブロックを吸収し、`画数 × 10` のコインを獲得する。

#### IncineratorNode（焼却炉）
- プレイヤーが配置可能なゴミ箱。
- 不要なブロックや詰まったブロックを吸収して破棄する。（コインは得られない）

### 3.3 コンベアベルト（ConveyorBelt）

- 4方向: `Up`, `Right`, `Down`, `Left`
- 矢印スプライトでコンベアの方向を視覚的に表示
- アイテムを次マスのコンベア・または施設へ送り出す

## 5. コアシステム

### 5.1 ティックシステム（ItemManager）

- `tickInterval` : 0.8秒ごとに全アイテムの移動をシミュレーション。
- 滑らかな移動： 各アイテムは目的地まで `SmoothStep` アニメーションでトランジション。
- アイテムが施設のあるセルへ到達した際、`Facility.CanAcceptItem()` を使って施設の許容状態を問い合わせる。受け入れ可能であれば `OnItemArrived()` で施設内に取り込まれる。

### 5.2 合体カラーシステム

アイテムの `MergeCount` に応じて色が変化（最大8段階）。
0:基本(ランダム), 1:青系, 2:緑系, 3:黄系, 4:紫系, 5:オレンジ, 6:シアン, 7+:白

## 6. 経済 / データ

### 6.1 コイン獲得計算式
```
獲得コイン = max(1, 画数) × 10
```
- メインハブ搬入時のみ得られる。
- 焼却炉へ入れた場合は0。

- `Assets/Resources/KanjiRecipe.csv` から合体レシピを解決。
- 合成機側でのロジック変更により、"日" + "月" = "明" という定義に対し、"月" + "日" の順で入力されても正しく合体が可能になった。
- 存在しない組み合わせ（Permutationのどちらでも見つからない場合）は合体失敗扱い。
